MANUAL COMPLETO DO CUMA
Versão: 1.100.30

1. Estrutura do pacote
O pacote limpo contém a pasta Repositorio_GitHub. O conteúdo dessa pasta deve ser copiado para a raiz do repositório GitHub.

Estrutura esperada no repositório:
- cuma.py
- cuma_updater.py
- requirements.txt
- .github/workflows/build_multiplataforma.yml
- arquivos necessários/
- scripts/
- updates/
- docs/
- README.md
- NOTAS_RELEASE.md
- AUDITORIA_DEBUG.md

2. Build pelo GitHub Actions
Após commit e push, abra o GitHub, vá em Actions e execute Build CUMA Multiplataforma.
O workflow gera:
- CUMA_windows.zip
- CUMA_linux.tar.gz
- CUMA_macos.zip

3. Correção 1.100.30
A 1.100.30 corrige o Build Windows quando o script scripts/preparar_manifesto_release.py não é encontrado no layout usado pelo GitHub Actions.
O manifesto stable.json é útil, mas a falha dele não deve impedir o upload do artifact. Por isso, o script agora registra aviso e termina com sucesso quando o ZIP final já foi criado.

4. Atualização
O CUMA usa updates/stable.json para detectar versão nova e escolher o pacote certo por plataforma.
Depois de publicar os artifacts na release Stable, atualize os links e SHA256 no stable.json.

5. Logs
- CUMA.log: log geral do programa.
- erro.txt: erros do programa.
- CUMA_update.log: log do atualizador.
- CUMA_backup_anterior: backup único da versão anterior.
